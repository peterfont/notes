# 可靠的 Agent Loop 由什么组成？

## 核心结构

一个可靠的 Loop，至少包含四个阶段：Discover、Plan、Execute、Verify，并且循环进行，直到证据证明任务真的完成。

## 完整整理稿

AI 明明说文章已经发好，你打开后台，草稿箱却是空的。这不只是操作失误，而是任务没有形成闭环。专业上，这叫 Loop Engineering。

以发布一篇文章为例，一个 Loop 可以拆成四个阶段：

- **Discover：发现状态。** 账号登录了吗？素材齐了吗？现在是否已经存在草稿？
- **Plan：规划下一步。** 先上传封面，再创建草稿，明确每一步的输入和验收方式。
- **Execute：执行动作。** 真正上传封面、写入正文、创建草稿。
- **Verify：验证结果。** 重新打开后台，确认结果是否真实存在。验证不通过，就把错误带回下一轮。

要让 Loop 知道什么时候结束，还需要两个概念。

**Acceptance Criteria（验收条件）**：例如草稿箱中有正确标题，正文可以打开，封面正常显示。

**Stop Condition（停止条件）**：例如连续失败三次就暂停，并报告卡点。验收条件防止假成功，停止条件防止 AI 无限重试、浪费时间和积分。

同一个 AI 既执行又检查，容易产生确认偏误。因此在更严谨的工程里，可以拆成两个角色：执行者负责上传封面、创建草稿；验收者重新打开后台，只按照预先定义的证据判断结果。

Loop 还需要 External State（外部状态）。不要只依赖聊天记忆，可以准备三个文件或记录：任务清单，记录做到哪一步；成果文件，保存正文、封面和草稿链接；错误日志，记录失败原因。每完成一步就写入状态，这样即使中断，也能从最后一个已确认步骤继续。

可靠的 Loop，可以概括为：状态、动作、验证，再加上可恢复的记忆。

## 落地方法

选一个重复任务，写下开始前要观察什么、执行顺序是什么、用什么证据验收、失败几次就停止。然后让一个 AI 执行，让另一个角色验收。
