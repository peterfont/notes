# Loop 真正的瓶颈是验证

## 核心观点

验证不是最后补上的一步，而是决定结果能不能交付的核心能力。越早验证、越小批次，错误成本越低。

## 完整整理稿

AI 十分钟生成三十个商品页，为什么你还要花两小时返工？因为最贵的从来不是生成慢，而是最后才发现三十个页面用了错误价格。

生成速度比检查速度快时，错误会像复制粘贴一样扩散。一个错误价格进入模板，三十个页面就会一起出错；等全部生成完才检查，返工范围已经被放大。

因此，验证要迁移到每个关键动作之后，让错误停在最小批次里，而不是等到最后集中爆炸。

一个可靠的 Loop 至少要留下三层证据：

1. **结果检查：** 价格、标题、图片是否符合目标；
2. **过程检查：** 来源、权限、禁用词和执行路径是否合规；
3. **状态检查：** 后台是否真的发布，之后还能不能通过 ID 重新查到。

硬检查不接受“应该没问题”，而是要求每个关键项输出明确的 Pass 或 Fail：价格是否一致、接口状态码是否正确、发布后的页面能否打开。

验证器的职责，是主动寻找能够推翻成功的证据，而不是替执行者写一份漂亮总结。

同时还要设置成本闸门：先规定哪些证据齐全才算交付，再设置最大重试次数、预算上限和每批页面数量。如果同一个问题连续失败三次，就触发 Circuit Breaker，停止继续消耗 Token，转人工定位。

停下来不是失败，而是在保护系统不把小错变成大障碍。生成负责给你答案，验证负责让答案能够承担后果。
